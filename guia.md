# Guía de Clase 2: Prompts efectivos

## ¿Qué es un prompt?

Un prompt es la instrucción que damos a GitHub Copilot para que nos ayude a generar código o explicaciones.

## Principios básicos para un buen prompt

**Ser específico**: indicar claramente lo que quieres lograr.

- Dar **contexto**: explicar el lenguaje, framework o archivo.
- Usar **pasos**: dividir una tarea compleja en pasos pequeños.
- Revisar siempre el resultado y no aceptar a ciegas.

## Ejemplo

❌ Mal prompt: "haz una función"
✅ Buen prompt: "Crea una función en JavaScript que reciba un array de números y devuelva el promedio."
