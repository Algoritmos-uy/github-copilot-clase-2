## ¿Qué es un prompt?

Un **prompt** es la instrucción o comentario que escribimos para que GitHub Copilot nos ayude a generar código automáticamente.

Piensa en el prompt como una **conversación con un asistente muy inteligente**:
- Le dices qué necesitas
- Él te ayuda a escribir el código
- Tú revisas si está correcto

### Formas de usar prompts:
- **Comentarios en el código**: `// Crea una función que...`
- **En Copilot Chat**: Conversaciones más largas
- **Inline**: Mientras escribes código

---

## ¿Por qué son importantes los prompts?

Un buen prompt es la diferencia entre:
- ✅ Obtener código útil y correcto
- ❌ Obtener código confuso o incorrecto

**Recuerda**: Copilot es muy inteligente, pero necesita instrucciones claras para ayudarte mejor.

---

## Principios básicos para un buen prompt

### 1. **Ser específico**
Indica claramente lo que quieres lograr.

### 2. **Dar contexto**
Explica:
- En qué lenguaje de programación
- Qué tipo de aplicación
- Qué problema estás resolviendo

### 3. **Usar pasos simples**
Si es complejo, divídelo en tareas pequeñas.

## Ejemplos prácticos

### Ejemplo 1: Función básica
❌ **Mal prompt**: 
```javascript
// haz una función
```

✅ **Buen prompt**: 
```javascript
// Crea una función en JavaScript que reciba un nombre como parámetro y devuelva un saludo personalizado
```

### Ejemplo 2: Componente React
❌ **Mal prompt**: 
```javascript
// componente contador
```

✅ **Buen prompt**: 
```javascript
// Crea un componente React que muestre un contador inicializado en 0 y un botón para incrementarlo
```

### Ejemplo 3: Estilos CSS
❌ **Mal prompt**: 
```css
/* centrar div */
```

✅ **Buen prompt**: 
```css
/* Crea CSS para centrar un div horizontal y verticalmente en la pantalla usando Flexbox */
```

## Resumen

**Un buen prompt tiene**:
- Descripción clara de lo que quieres
- Contexto del lenguaje/tecnología
- Pasos simples y específicos

**Recuerda**: La práctica hace al maestro. Mientras más uses Copilot con buenos prompts, mejor será tu experiencia de programación.
